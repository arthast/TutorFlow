Ключевые правила (UX → backend)
priority_high
Загрузка чека НЕ меняет долг. Баланс пересчитывается только при confirm чека преподавателем.
block
Реальных платежей нет — никакой эквайринг. Только ручное подтверждение.
visibility
Ученик видит долг только на чтение; правки баланса — у преподавателя (корректировки ±).
bolt
Realtime: счётчики непрочитанного, статусы онлайн и тосты приходят push-ом.
Экран → ключевые операции бэкенда
Экран
Операции
Ученики / Карточка
list / create student (email + temp pw), get profile, update notes & fields, adjust balance (±, comment)
Занятия
list (upcoming / past), create, complete (→ charge), cancel, reschedule, reactivate
Домашние задания / Проверка
create HW (files, deadline), list, get submissions, review (accept / revise + comment), download files
Чеки / Оплата
student: declare payment (amount + file). teacher: list pending, confirm (→ payment entry), reject (reason)
Финансы
aggregate debts, per-student ledger, recent operations, export
Чат / Уведомления
conversations, messages + attachments, mark-read, presence, notifications — через WebSocket/SSE
info
Прототип — статический: данные хранятся в стейте каждого экрана (демо-значения на русском), действия дают тосты и оптимистично обновляют UI. Для продакшена: заменить локальный стейт на вызовы API-gateway, подключить realtime-канал для чата и уведомлений, файлы — через загрузку/скачивание вложений. Палитра, типографика и компоненты — в «Дизайн-системе».
